# BabyLog v0.2 — 核心体验升级（已完成）

## 完成状态

v0.2 全部 3 个模块、18 个任务已完成，包含：
- 模块一（交互优化）：8 项 — 操作反馈、流程优化、时间线编辑删除、设置页重构、状态栏适配、i18n、用户资料、E2E 验证
- 模块二（核心记录）：5 项 — 7 种事件丰富表单、喂奶/睡眠计时器、今日总览仪表盘、深色模式、E2E 验证
- 模块三（记忆与照片）：5 项 — R2 照片上传、宝宝档案增强、事件附带照片、里程碑记录、E2E 验证

E2E 测试覆盖：66 个流程、385 张截图。

## 目标

让 BabyLog 从"能用"变成"好用"——交互打磨到位、记录信息丰富、支持照片和里程碑，为下一期家庭分享打下内容基础。

## 用户故事

> 作为一个新手妈妈，我希望：
> - 凌晨喂奶时 app 不刺眼（深色模式）
> - 一键开始计时，喂完自动记录时长（计时器）
> - 记录喂奶的量、方式、时长，而不只是"喂了一次"（丰富记录）
> - 拍下宝宝第一次翻身的照片，记录这个里程碑（照片 + 里程碑）
> - 操作后有明确反馈，不用猜是否成功（交互优化）
> - 我看得懂中文，但老公是外国人，他需要英文界面（中英文切换）

## 功能模块

### 模块一：交互优化

**背景**：E2E 截图审计（11 个流程，77 张截图）发现 5 个已知问题 + 6 个新问题。

#### 1.1 操作反馈补全
- 添加宝宝成功：Toast "添加成功" + 列表刷新
- 记录事件成功：Toast "记录成功" + "继续记录"/"查看时间线"选择
- 删除操作（事件/宝宝）：二次确认弹窗
- 退出登录：二次确认弹窗（当前无确认，存在误触风险）
- 网络错误：统一错误 Toast

#### 1.2 流程优化
- **注册 token 页**：加大 token 字号 + 醒目复制按钮 + "请妥善保存，丢失无法找回"安全提醒
- **快速记录**：完成后智能跳转 + 成功提示；只有一个宝宝时自动选中
- **快速记录空状态**：增加"去添加宝宝"按钮（当前仅文字无法点击）
- **登录页**：底部"去注册/创建"引导文字加大
- **首页**：增加上次记录时间提示 + 宝宝头像占位符

#### 1.3 时间线编辑/删除
- 事件卡片长按弹出操作菜单（编辑/删除）
- 编辑：预填表单 → `PATCH /events/:id`
- 删除：二次确认 → `DELETE /events/:id`

#### 1.4 设置页分区重构
- 按分组排列：应用设置 / 宝宝管理 / 账号与安全
- "添加宝宝"改为独立页面或底部弹窗
- 为后续功能预留位置（家庭成员、数据导出、深色模式）

#### 1.5 系统状态栏适配
- 统一 `app.json` 设置与 app 内深色 UI 的状态栏风格

#### 1.6 用户个人资料（本地版）

当前系统只有 family token，没有"用户"身份。在 v0.3 family_members 之前，先做本地版个人资料，为时间线"谁记录的"功能占位。

- 设置页"账号与安全"区域增加"我的称呼"和"我的头像"
- 称呼：文本输入（如"妈妈"、"爸爸"）
- 头像：可拍照/选择相册（复用 3.1 照片上传基础设施，若 3.1 未完成则先存本地文件 URI）
- 存储到 AsyncStorage（本地持久化）
- 时间线事件卡片显示记录人称呼（如"妈妈 · 3/29 10:30"）
- v0.3 迁移计划：family_members 表就绪后，将本地称呼和头像同步到服务端

#### 1.7 中英文国际化（i18n）

App 需支持中文和英文两种语言，用户可在设置页切换，也可跟随系统语言。

**技术方案：**
- 使用 `expo-localization` 获取系统语言 + `i18next` / `react-i18next` 作为 i18n 框架
- 新增 `apps/mobile/src/i18n/` 目录，包含：
  - `index.ts` — i18n 初始化配置
  - `locales/zh.json` — 中文翻译文件
  - `locales/en.json` — 英文翻译文件
- 所有屏幕和组件中的硬编码中文文案替换为 `t('key')` 调用
- 设置页增加"语言"切换项（中文 / English / 跟随系统）
- 语言偏好持久化到 AsyncStorage
- API 返回的固定文案（如错误信息）也需要支持 `Accept-Language` header 或由客户端根据 error code 映射

**需要翻译的内容范围：**
- 所有屏幕标题和按钮文案
- Toast 提示文案
- 事件类型名称（喂奶/睡眠/尿布…）
- 里程碑预置模板名称
- 表单标签和占位符文本
- 错误提示信息
- 确认弹窗文案

**注意事项：**
- 此任务应在模块二和模块三**之前**完成，这样新功能的文案从一开始就用 i18n key，不需要返工
- 里程碑模板等内容如果在 i18n 之后开发，可直接用翻译 key

### 模块二：核心记录增强

#### 2.1 丰富事件记录详情

每种事件类型有专属表单字段：

- **喂奶 feed**：方式（母乳/配方奶/混合）、量(ml)、时长(分钟)、左右侧
- **睡眠 sleep**：开始/结束时间、时长（自动算）、质量（好/一般/差）
- **尿布 diaper**：类型（湿/干/混合）、备注
- **便便 poop**：颜色（黄/绿/棕/异常）、质地（正常/稀/硬）、量
- **体重 weight**：体重(kg)、备注
- **辅食 solid**：食物名称、量、过敏反应（无/轻微/严重）
- **疫苗 vaccine**：疫苗名称、接种机构、下次接种提醒日期

需同步更新 `packages/shared-types` 类型定义 + API payload 验证。

#### 2.2 喂奶/睡眠计时器
- 快速记录页"开始计时"按钮
- App 顶部常驻计时条（类似通话中绿色横条）
- 后台计时支持（app 切后台继续）
- 计时结束自动填充时长到事件记录

#### 2.3 今日总览仪表盘
- 新增 API：`GET /stats/today?babyId=xxx`
- 替换首页导航卡片为数据仪表盘：今日喂奶次数+总量 / 睡眠时长 / 上次换尿布距今 / 上次喂奶距今
- 多宝宝时顶部切换
- 保留快速入口按钮

#### 2.4 深色模式
- 扩展 `theme.ts` 为 `lightTheme` + `darkTheme`
- 系统跟随 + 设置页手动切换
- 全屏幕适配（不硬编码颜色）

### 模块三：记忆与照片

> 这是通往 v0.3 家庭分享的基础——没有照片和里程碑，就没有值得分享的内容。

#### 3.1 照片上传系统
- 存储：Cloudflare R2
- API：`POST /upload`（signed URL 或 multipart）
- 前端：`expo-image-picker` 选择/拍照 + 图片压缩
- 图片展示：缩略图 + 点击全屏查看

#### 3.2 宝宝档案增强
- DB 扩展 `babies` 表：`gender`、`height`、`blood_type`、`allergies`、`avatar_url`、`notes`
- 宝宝详情页：头像上传、完整信息编辑
- 设置 & 首页显示宝宝头像

#### 3.3 事件附带照片
- 快速记录增加"添加照片"（可选，最多 9 张）
- `events.payload` 增加 `photoUrls` 数组
- 时间线显示照片缩略图网格 + 全屏查看

#### 3.4 里程碑记录
- 新事件类型 `milestone`（复用 events 表，统一时间线）
- 预置模板：第一次微笑、翻身、坐、爬、站、走、叫妈妈/爸爸、长第一颗牙、第一次吃辅食…
- 自定义里程碑 + 必选配照片 + 日期 + 备注
- 时间线中里程碑用特殊卡片样式突出
- 时间线筛选：全部 / 日常 / 里程碑
