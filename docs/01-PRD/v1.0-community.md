# BabyLog v1.0 — 分享广场（远期愿景）

## 目标

从私有家庭记录走向公开育儿社区——让不同家庭之间可以分享育儿经验、互相帮助，形成一个有温度的育儿内容平台。

## 用户故事

> 作为一个新手妈妈，我希望：
> - 宝宝 6 个月了不知道怎么加辅食，想看看别的妈妈怎么做的（经验参考）
> - 把宝宝第一次走路的视频分享出去，获得其他家长的祝贺（社交互动）
> - 搜索"4 个月宝宝睡眠"看看同月龄的宝宝都睡多少（同龄参考）
> - 发现一个很好的辅食食谱，想收藏下来以后用（收藏）

## 功能模块

### 模块一：用户账号体系升级

v0.1~v0.3 使用 family token，v1.0 需要升级为正式用户账号：
- 手机号/邮箱注册登录
- OAuth 登录（微信、Apple）
- 用户个人主页：昵称、头像、宝宝月龄、发布的内容
- 隐私设置：哪些内容对外可见，哪些仅家庭可见
- 向前兼容：已有 family token 用户可绑定手机号升级

### 模块二：内容发布系统

- 从私有时间线"分享到广场"（一键发布）
- 独立创建图文帖子：标题 + 正文 + 最多 9 张图
- 短视频：录制或上传，限时长 60s
- 发布时可选择：实名 / 匿名 / 仅显示昵称
- 话题标签：#辅食食谱 #睡眠训练 #成长记录 #育儿好物 #求助
- 内容审核：敏感词过滤 + 社区举报机制

### 模块三：广场信息流

- App 新增"发现"Tab
- 信息流展示其他家庭的公开分享（图文/视频混排）
- 分类浏览：喂养经验 / 睡眠训练 / 辅食食谱 / 成长里程碑 / 育儿好物 / 求助
- 搜索：按关键词、标签、宝宝月龄
- 推荐算法：根据自家宝宝月龄、浏览偏好、关注话题推荐内容
- 热门/最新排序切换

### 模块四：社交互动

- 点赞、收藏帖子
- 评论（支持楼中楼回复）
- 关注其他用户/家庭
- 消息通知：被赞、被评论、被关注、被 @ 提及
- 私信（可选，需考虑安全性）

## 技术变更摘要

- 用户账号体系：新增 `users` 表、OAuth 集成、session/JWT 管理
- 内容系统：`posts` 表、`comments` 表、`likes` 表、`follows` 表
- 视频：Cloudflare Stream 或 R2 + 转码
- 信息流：排序/推荐服务（可从简单的时间序 + 月龄匹配开始）
- 内容审核：基础敏感词 + 举报队列
- App 导航：从单栈导航升级为底部 Tab 导航（首页 / 发现 / + / 消息 / 我的）

## 风险与思考

- **隐私**：涉及宝宝照片的公开分享，隐私保护是核心——默认私有，分享需主动操作，支持随时撤回
- **内容安全**：必须有审核机制，防止不良内容
- **冷启动**：社区初期内容少，可考虑官方账号发布优质育儿内容作为种子
- **商业化**：育儿好物推荐、品牌合作、会员增值（无广告、更大存储空间）

## 不在本期范围（后续迭代）
